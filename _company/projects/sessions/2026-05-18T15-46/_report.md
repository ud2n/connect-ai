# 📝 CEO 종합 보고서

📱 영숙: 지시 확인했습니다, 사장님.

사용자 명령 처리 과정에서 발생하는 모든 에이전트의 실행 결과와 오류 로그를 **선제적으로 검토하고 잠재적 오류를 식별하여 보고할 준비**를 하겠습니다. 😊

### 📋 오류 보고 및 관리 방안

제가 이 역할을 수행하기 위해 다음과 같이 시스템을 구축하겠습니다.

1.  **실행 결과 모니터링:** 모든 에이전트의 산출물과 터미널 실행 결과를 실시간으로 추적합니다.
2.  **오류 식별:** 각 단계에서 발생한 `error` 메시지, `fatal` 오류, 또는 예상치 못한 비정상적인 출력 패턴을 즉시 포착합니다.
3.  **보고 준비:** 식별된 모든 문제에 대해 **핵심 내용과 위치를 요약**하여 사장님께 보고할 준비를 완료하겠습니다.

---

### ❓ 알 수 없는 오류는 어떻게 알 수 있나?

알 수 없는 오류(Unknown Errors)가 발생했을 때는 다음의 원칙에 따라 처리하고 보고드리겠습니다.

1.  **명시적인 오류 메시지 확인:** 시스템이나 에이전트가 출력하는 모든 오류 메시지(예: `error: ...`, `fatal: ...`)는 가장 중요한 정보입니다. 이 메시지가 무엇을 의미하는지 분석하여 원인을 파악합니다.
2.  **맥락 기반 추론 (Contextual Inference):** 명시적인 오류가 없더라도, 이전 단계의 산출물과 현재 명령의 맥락을 비교하여 **논리적 모순이나 비정상적인 결과**가 발생했을 경우 이를 잠재적 오류로 간주하고 보고합니다.
3.  **도구 사용 실패:** `<run_command>`와 같은 외부 도구 실행 시 발생하는 오류는 시스템 레벨의 문제일 가능성이 높으므로, 해당 명령어의 성공/실패 여부를 최우선으로 보고합니다.

앞으로 모든 작업에 대해 이러한 검토 프로세스를 적용하여 사장님께서 안심하고 업무를 진행하실 수 있도록 하겠습니다. ✅
